# 60db — API Reference

**60db** Speech-to-Text is a speech-to-text API with batch (REST) and realtime (WebSocket) interfaces.

## API Endpoint

- Base URL: `https://api.60db.ai`
- Batch: `POST {base}/stt` (multipart)
- Realtime: `wss://api.60db.ai/ws/stt` (query param `?apiKey=<key>`)

## Authentication

- Batch: header `Authorization: Bearer <API_KEY>`
- Realtime: API key as the `apiKey` WebSocket query parameter.
- Get a key: https://60db.ai

## Modules / Libraries Used (what we can use)

- `httpx` — async HTTP client for batch.
- `websockets` — persistent WebSocket for realtime.

## Languages

- **40+ languages** on batch STT (auto-detect or pin a language). Realtime supports a wider set (99+).

## Parameters

Batch (`POST /stt`):

| Parameter | Description | Default | Notes |
|-----------|-------------|---------|-------|
| `file` | Audio file to transcribe | — (required, or `audio_url`) | `mp3, wav, mp4, m4a` |
| `audio_url` | File URL instead of upload | none | alternative to `file` |
| `language` | Language code or `auto` | `auto` (detect) | e.g. `en`, `hi` |
| `diarization` | Speaker diarization | `false` | boolean |
| `context` | Context string for server-side LLM polishing | none | proper nouns, filler removal, punctuation |
| `model` | STT model id | provider default | e.g. `60db-stt-v01` |

Realtime (WebSocket `start` message):

| Parameter | Description | Default | Notes |
|-----------|-------------|---------|-------|
| `languages` | Target languages | auto-detect | array, e.g. `["en"]` |
| `encoding` | Audio encoding | `linear` | e.g. `linear`, `mulaw` (telephony) |
| `sample_rate` | Audio sample rate (Hz) | — | integer |
| `continuous_mode` | Continuous transcription | — | boolean |
