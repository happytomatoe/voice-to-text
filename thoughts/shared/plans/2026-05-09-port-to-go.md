# Port groq-voice to Go Implementation Plan

## Overview

Port the existing Python voice-to-text application to Go, maintaining the same functionality: real-time audio recording with FFT visualization, Groq Whisper API transcription, and text output (type or clipboard).

## Current State Analysis

Python app with:
- `sounddevice` for audio recording
- `numpy` for FFT analysis and audio processing
- `groq` SDK for Whisper API transcription
- curses-based TUI with real-time frequency bars
- subprocess calls to `wtype`/`xclip` for output

## Desired End State

Pure Go application with:
- Go PortAudio bindings (`github.com/cgos存活`)
- Same TUI with FFT visualization
- Groq API client (standard `net/http`)
- Single binary with cgo linking to PortAudio `.so`

## What We're NOT Doing

- GUI version (keeping TUI)
- Cross-platform audio backends beyond Linux
- WASM/web targets
- Python integration layer

### Changes Required:

#### Config/logging setup
**File**: `cmd/groq-voice/main.go`
```go
import "log/slog"

// Log to both stderr and file (~/.local/share/groq_voice/groq_voice.log)
func setupLogging() *slog.Logger {
    f, err := os.OpenFile(
        os.ExpandEnv("$HOME/.local/share/groq_voice/groq_voice.log"),
        os.O_CREATE|os.O_APPEND|os.O_WRONLY, 0644,
    )
    if err != nil {
        slog.Warn("failed to open log file", "err", err)
    }
    return slog.New(slog.NewJSONHandler(f, nil))
}
```
